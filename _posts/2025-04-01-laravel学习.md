# laravel学习笔记

## 解决laravel8.x用户认证vite指令不生效问题
### 问题背景：
在Laravel Framework 8.83.29学习过程中，按照文档启用用户认证，使用的laravel/jetstream 扩展包：
```bash
composer require laravel/jetstream

// 使用 Livewire 栈安装 Jetstream...
php artisan jetstream:install livewire

// 使用 Inertia 栈安装 Jetstream...
php artisan jetstream:install inertia
```
### 问题描述：

但是在访问xxx/register时，出现了问题：
页面 @vite 不起作用，页面表现就是 css，跟 js 没有加载。

以下这行指令 
```html
@vite ([‘resources/css/app.css’, ‘resources/js/app.js’])
```
在页面中没有被解析,直接以文本的形式显示出来了。

### 解决方法：
在根目录下执行以下命令：
```bash
composer require laravel/breeze:1.9.4
php artisan breeze:install
```

### 原因分析：
在 Laravel 8.83.29 中，@vite 指令是用于在 Laravel 中引入 Vite 打包后的资源文件的。 但是，在 Laravel Jetstream 和 Laravel Breeze 中，这两个扩展包已经包含了 Vite 的支持，因此，@vite 指令在 Laravel Jetstream 和 Laravel Breeze 中已经不需要了。 因此，在使用 Laravel Jetstream 和 Laravel Breeze 时，@vite 指令是无效的。其实这个目录在前段时间学习时是成功过的，隔了很长一段时间，就没有使用过了。今天再次按文档学习时，就出现了这个问题。
