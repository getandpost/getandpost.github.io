# AI Agent / AI 调试工程师系统学习计划

**制定时间**: 2026-06-16  
**目标**: 从零系统学习 AI Agent 工程、LLM 应用开发、RAG、工具调用、MCP、工作流编排、评测调试、部署运维、安全控制，最终产出可展示项目集。  
**周期建议**: 24 周，约 6 个月。  
**每日投入**: 1.5-3 小时。  
**学习原则**: 理论 30%，实操 50%，项目 20%。每个阶段必须有可运行作品。

---

# 一、总目标

最终达到以下能力：

1. 能独立开发一个基于大模型 API 的 AI 应用。
2. 能用 RAG 构建企业知识库问答系统。
3. 能封装工具调用，让 Agent 调 API、读文件、查数据库、执行脚本。
4. 能用 LangGraph / Dify / Coze / AutoGen / CrewAI 设计 Agent 工作流。
5. 能调试 Agent：看 trace、定位失败、优化 Prompt、工具、RAG、状态流。
6. 能做 Agent 评测：测试集、指标、回归测试、A/B 对比。
7. 能部署生产级 Agent：Docker、日志、权限、安全、监控告警。
8. 能输出 3-5 个简历级项目。

## 阶段 2：LLM API 与 Prompt Engineering（第 5-7 周）

### 目标

能调用大模型 API，写稳定 Prompt，让模型按格式输出。

### 理论

- LLM 基本原理
- Token 与上下文窗口
- System Prompt / User Prompt
- Temperature
- JSON 输出约束
- few-shot
- Chain-of-Thought 的正确使用方式
- 幻觉问题
- Prompt Injection 基础

### 实操任务

#### Task 1：LLM Chat Wrapper

写一个统一调用层：

```python
class LLMClient:
    def chat(messages): ...
    def structured_chat(schema): ...
```

支持：

- DeepSeek
- OpenAI compatible API
- 通义 / 豆包，可选

#### Task 2：结构化输出

让模型输出：

```json
{
  "summary": "...",
  "risk_level": "low|medium|high",
  "actions": ["..."]
}
```

要求：

- 校验 JSON
- 失败重试
- 格式错误自动修复

#### Task 3：Prompt 版本对比

同一个任务写 3 个 Prompt，比较输出差异。

### 输出项目

**项目 2：投资资讯摘要助手**

功能：

- 输入一段财经新闻
- 输出摘要、利好/利空、风险、操作建议

### 验收标准

- 能调用 LLM API

- 能知道 Prompt 为什么失败

## 阶段 3：Function Calling / Tool Use（第 8-10 周）

### 理论

#### Task 2：Tool Router

实现一个简单 Agent：

用户问：“查一下 BTC 价格并分析。”

Agent 自动：

1. 判断需要行情工具
2. 调 `get_price`
3. 把结果交给 LLM 分析
4. 输出结论

#### Task 3：安全确认

所有写文件、下单、发消息类工具，必须加人工确认。

### 输出项目

**项目 3：行情分析 Agent**

## 阶段 5：Agent 工作流与 LangGraph（第 15-17 周）

### 目标

从“一个模型调用工具”升级到“可控工作流 Agent”。

### 理论

- ReAct
- Planner-Executor
- 状态机
- DAG
- LangGraph 核心概念
- 节点、边、状态
- checkpoint
- human-in-the-loop
- 多步任务失败恢复

### 实操任务

#### Task 1：LangGraph 入门

实现流程：

```text
用户目标 → 计划节点 → 工具节点 → 分析节点 → 自检节点 → 输出节点
```

#### Task 2：带状态的 Agent

状态包括：

- 用户目标
- 已调用工具
- 中间结果
- 错误信息
- 最终输出

#### Task 3：失败重试

工具失败时：

- retry 2 次
- fallback
- 输出错误报告

#### Task 4：人工确认节点

高风险操作进入人工确认：

- 下单
- 删除文件
- 发消息
- 修改数据库

### 输出项目

**项目 5：交易监控工作流 Agent**

功能：

- 能用 LangGraph 做状态流

# 十二、注意事项

1. 不要一开始就卷大模型训练。
2. 不要沉迷框架，先理解 Agent 为什么失败。
3. 不要只会拖 Dify / Coze，必须会代码。
4. 不要忽略评测，Agent 没有 eval 就是在玄学调参。
5. 不要忽略安全，能调用工具的 Agent 有真实破坏力。
6. 不要只做玩具项目，要做真实 API、真实日志、真实权限、真实异常。
